You are a senior DevOps and Go engineer. Integrate the Lovable-generated frontend repository into the main clockwise-like backend repository.

Main repo:     git@github.com:Enach/clockwise-like.git
Frontend repo: https://github.com/Enach/smart-calendar-flow

The frontend was generated by Lovable and lives in its own GitHub repo.
The goal is to pull its source into the main repo under frontend/ using git subtree,
so the full stack (backend + frontend) builds together with a single `docker compose up --build`.

## Context

- T-01 created a placeholder frontend scaffold at frontend/ (Dockerfile, package.json, vite.config.ts, nginx.conf, and a stub src/)
- The Lovable repo (smart-calendar-flow) contains the real React source code that replaces that stub
- After this task, `frontend/src/` must contain the Lovable code, not the placeholder
- The frontend/Dockerfile, vite.config.ts, and nginx.conf from T-01 must be preserved as-is

## Step 1 — Add the Lovable repo as a git subtree remote

In the clockwise-like repo:

```bash
git remote add lovable-frontend https://github.com/Enach/smart-calendar-flow
git fetch lovable-frontend
```

## Step 2 — Merge the Lovable repo's src/ into frontend/src/

Use git subtree to pull only what is needed. The Lovable repo likely has its source at the root (src/, package.json, etc.).

Strategy: use `git subtree add` to pull the entire Lovable repo into a temporary path, then move its contents into frontend/.

```bash
# Pull the Lovable repo into a temp directory inside the repo
git subtree add --prefix=_lovable_tmp lovable-frontend main --squash
```

Then:

- Copy `_lovable_tmp/src/` → `frontend/src/` (replacing the placeholder)
- Merge `_lovable_tmp/package.json` dependencies into `frontend/package.json`:
    - Keep ALL deps from the Lovable package.json
    - Keep the existing devDependencies from T-01's package.json (vite, typescript, tailwindcss, postcss, autoprefixer)
    - The `name`, `version`, `scripts` fields should come from T-01's package.json (preserve the existing build scripts)
- Do NOT copy from _lovable_tmp: Dockerfile, vite.config.ts, index.html, nginx.conf, .env* files
- Remove the _lovable_tmp/ directory after copying

```bash
git rm -r _lovable_tmp
```

## Step 3 — Verify the frontend builds

```bash
cd frontend
npm install
npm run build
```

Fix any build errors:

- If vite.config.ts has path alias issues (e.g. @/ imports), ensure vite.config.ts has the correct `resolve.alias` pointing to `src/`
- If tailwind.config.js content paths don't match, update to include `./src/**/*.{ts,tsx}`
- If index.html references are broken, check that `<script type="module" src="/src/main.tsx">` is present

## Step 4 — Verify the full stack builds with Docker Compose

```bash
cd .. # back to repo root
docker compose build
```

Both `backend` and `frontend` services must build without error.

## Step 5 — Add a Makefile target to pull future Lovable updates

Add to the existing Makefile (create one if it doesn't exist):

```makefile
.PHONY: update-frontend

## Pull latest changes from the Lovable frontend repo into frontend/src/
update-frontend:
	git fetch lovable-frontend
	git subtree pull --prefix=_lovable_tmp lovable-frontend main --squash
	cp -r _lovable_tmp/src/. frontend/src/
	@echo "Merging package.json dependencies..."
	node -e " \
	  const fs = require('fs'); \
	  const base = JSON.parse(fs.readFileSync('frontend/package.json')); \
	  const lovable = JSON.parse(fs.readFileSync('_lovable_tmp/package.json')); \
	  base.dependencies = { ...lovable.dependencies, ...base.dependencies }; \
	  fs.writeFileSync('frontend/package.json', JSON.stringify(base, null, 2) + '\n'); \
	  console.log('Done merging dependencies'); \
	"
	git rm -r _lovable_tmp
	git add frontend/
	git commit -m "chore: sync frontend from Lovable (smart-calendar-flow)"
	@echo "Frontend updated. Run 'docker compose build' to rebuild."
```
