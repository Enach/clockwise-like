# T-14 — Personal calendar → work calendar blockers

## Goal
Let users connect personal calendars (Google, Outlook, WebCal) and automatically create opaque "[Personal] Busy" blocker events in their work calendar so colleagues see them as unavailable.

## DB migration (`004_personal_calendars`)
```sql
CREATE TABLE personal_calendars (
    id SERIAL PRIMARY KEY,
    provider TEXT NOT NULL,  -- 'google', 'outlook', 'webcal'
    name TEXT, url TEXT, credentials_json TEXT, enabled BOOL, created_at TIMESTAMPTZ
);
CREATE TABLE personal_blockers (
    id SERIAL PRIMARY KEY,
    personal_calendar_id INTEGER REFERENCES personal_calendars(id) ON DELETE CASCADE,
    personal_event_id TEXT NOT NULL,
    work_event_id TEXT NOT NULL,
    created_at TIMESTAMPTZ,
    UNIQUE (personal_calendar_id, personal_event_id)
);
```

## calendar/personal_reader.go
`ReadPersonalEvents(ctx, *PersonalCalendar, start, end)` dispatches by provider:
- `webcal` → `NewWebcalClient`
- `outlook` → `NewOutlookClient` (token from `credentials_json`)
- `google` → `NewClient` (token from `credentials_json`)

## engine/personal_blocker.go
`PersonalBlocker.SyncAll(ctx)` — called every 30 min by cron in `main.go`:
1. Fetch events from personal calendar for next 14 days
2. Remove blockers whose personal event no longer exists (`deleteEvent` + `DeletePersonalBlocker`)
3. Create `[Personal] Busy` work-calendar events for new events that fall during work hours (`UpsertPersonalBlocker`)

`Preview(ctx, calID, start, end)` returns raw personal events without blocking.
`Sync(ctx, calID)` syncs a single calendar on demand.

## API routes
```
GET    /api/personal-calendars          — list
POST   /api/personal-calendars          — create {provider, name, url, enabled}
DELETE /api/personal-calendars/:id      — delete (cascades blockers)
GET    /api/personal-calendars/:id/preview — raw events for next 14 days
POST   /api/personal-calendars/:id/sync   — trigger immediate sync
```

## Cron
`@every 30m` in `main.go` via `robfig/cron/v3` — calls `PersonalBlocker.SyncAll`.
