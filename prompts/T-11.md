You are a senior Go engineer. Implement LLM provider switching via SSO (Bedrock + Azure OpenAI + Vertex AI + Ollama) in the Clockwise-like backend.

## Goal

Replace the direct API-key approach for all LLM providers with SSO-compatible cloud provider integrations. No API keys are stored anywhere.

| Provider | Models | Auth | Enterprise SSO path |
|:---|:---|:---|:---|
| **AWS Bedrock** | Claude opus-4-5, sonnet-4-5, haiku-4-5 | AWS IAM / STS | `aws sso login` — IAM Identity Center |
| **Azure OpenAI** | gpt-4o, gpt-4o-mini, gpt-4-turbo, gpt-35-turbo | Azure AD / Entra ID | `az login` browser flow, managed identity |
| **Google Vertex AI** | Gemini 2.5 Pro, 2.0 Flash, 1.5 Pro, 1.5 Flash | Google ADC | `gcloud auth application-default login`, Workload Identity |
| **Ollama** | llama3, mistral, qwen, phi3, … | None | Local only |

## DB changes

New migration `002_llm_providers.up.sql` adds columns to settings:
- AWS: aws_region, aws_profile, bedrock_model
- Azure: azure_endpoint, azure_deployment, azure_api_version
- GCP: gcp_project, gcp_location, vertex_model
- Ollama: ollama_base_url, ollama_model
