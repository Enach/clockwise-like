# T-16 — Conferencing provider integration (Zoom, Google Meet, Teams)

## Goal
Let users generate conference links (Zoom, Meet, Teams) as part of scheduling. The active provider is stored in `settings.conferencing_provider`.

## DB migration (`005_conferencing`)
```sql
ALTER TABLE settings ADD COLUMN conferencing_provider TEXT NOT NULL DEFAULT 'meet';
ALTER TABLE settings ADD COLUMN zoom_tokens TEXT NOT NULL DEFAULT '';
```

## conference/ package

### provider.go
```go
type Details struct { Provider, JoinURL, MeetingID, Password string }
type Provider interface { CreateMeeting(ctx, title, start, end) (*Details, error) }
```

### meet.go — `MeetProvider`
Returns empty `Details{Provider: "meet"}`. Google Meet links are provisioned
automatically by the Calendar API when `conferenceDataVersion=1` is passed on
event creation — no extra API call needed.

### zoom.go — `ZoomProvider`
POST `https://api.zoom.us/v2/users/me/meetings` with Bearer token.

### teams.go — `TeamsProvider`
POST `https://graph.microsoft.com/v1.0/me/onlineMeetings` with Bearer token.
Uses existing Microsoft OAuth token from `auth.LoadMicrosoftToken`.

### zoom_oauth.go
`ZoomAuthURL`, `ZoomExchangeCode` — standard OAuth2 code flow against `zoom.us/oauth/`.

### factory.go
`NewProvider(s *storage.Settings) (Provider, error)` dispatches on `s.ConferencingProvider`.
Teams short-circuits to `NewTeamsProvider(msToken.AccessToken)` in the handler.

## API routes
```
GET  /api/auth/zoom            — start Zoom OAuth (ZOOM_CLIENT_ID env required)
GET  /api/auth/zoom/callback   — exchange code, save tokens, redirect /?connected=true&provider=zoom
POST /api/conference/create    — { title, start, end } → { provider, joinUrl, meetingId?, password? }
```

## Env vars needed
`ZOOM_CLIENT_ID`, `ZOOM_CLIENT_SECRET`, `ZOOM_REDIRECT_URL`
