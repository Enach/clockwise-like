# T-22 — API Authentication: JWT + OAuth Google/Microsoft

## Goal
Add a complete JWT-based authentication layer to the backend. OAuth login (Google & Microsoft) issues a signed httpOnly cookie; all protected endpoints require a valid JWT.

## Architecture

1. User hits `GET /api/auth/google` or `GET /api/auth/microsoft`
2. Provider OAuth consent → callback exchanges code for token
3. Backend fetches user profile from provider, upserts into `users` table
4. Backend issues HS256 JWT (7-day TTL), sets it as an `auth_token` httpOnly cookie
5. All subsequent requests validated by `requireAuth` middleware (cookie or Bearer header)

## New files
- `backend/storage/migrations/006_auth.up.sql` — `users` table + `user_id` FK on `oauth_tokens`, `settings`, `focus_blocks`, `personal_calendars`
- `backend/auth/jwt.go` — `GenerateToken`, `ValidateToken`, `Claims`
- `backend/storage/users.go` — `UpsertUser`, `GetUserByID`, `User` struct
- `backend/api/handlers_me.go` — `GET /api/auth/me`, `POST /api/auth/logout`

## Modified files
- `backend/auth/token_store.go` — added `UpsertUserToken(db, userID, token)` and `LoadUserToken(db, userID)`; kept backward-compat aliases
- `backend/api/middleware.go` — added `requireAuth(jwtSecret)` + context helpers (`ctxUserID`, `userIDFromCtx`)
- `backend/api/handlers_auth.go` — Google callback fetches full userinfo, upserts user, issues JWT cookie; added `jwtSecret` field to `authHandlers`
- `backend/api/handlers_microsoft_auth.go` — Microsoft callback fetches Graph `/me`, upserts user, issues JWT
- `backend/api/routes.go` — `RegisterRoutes` accepts `jwtSecret string`; public routes: `/api/auth/{google,callback,microsoft,microsoft/callback,zoom,zoom/callback}`; protected: everything else
- `backend/main.go` — reads `JWT_SECRET` env var, passes to `RegisterRoutes`
- `docker-compose.yml` — added `JWT_SECRET=${JWT_SECRET:-}` env var

## API endpoints added
| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/api/auth/me` | JWT | Return current user `{id, email, name, avatarUrl}` |
| POST | `/api/auth/logout` | JWT | Clear `auth_token` cookie |

## Env vars
| Var | Purpose |
|---|---|
| `JWT_SECRET` | HS256 signing secret (leave empty to disable auth in dev) |

## Notes
- When `JWT_SECRET` is empty, auth middleware is bypassed (dev mode)
- Per-user data scoping on calendar/settings/focus queries is left for a follow-up; the `user_id` columns are migrated and available
- Microsoft callback fetches `GET /me?$select=id,mail,displayName` from Graph API
