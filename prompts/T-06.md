## Context
Smart scheduling finds the best time slots for a new meeting, given:
- Duration in minutes
- List of attendee emails
- Time range to search within (e.g., "this week")

It uses Google Calendar's freebusy API to find when everyone is free, then scores the slots.

## Task: Implement the Smart Scheduling engine

### engine/smart_schedule.go

**Structs:**
```go
type ScheduleRequest struct {
  DurationMinutes int
  Attendees       []string  // emails
  RangeStart      time.Time
  RangeEnd        time.Time
  Title           string
  Description     string
}

type SuggestedSlot struct {
  Start       time.Time
  End         time.Time
  Score       int
  Reasons     []string  // human-readable reasons for this score
}

type ScheduleSuggestions struct {
  Slots []SuggestedSlot  // top 3, sorted by score desc
}
```

**Algorithm:**
`func (e *SmartScheduler) Suggest(ctx, req ScheduleRequest) (*ScheduleSuggestions, error)`
1. Fetch freebusy for all attendees (including the user's own calendar) for the range
2. Compute intersection of free slots: a slot is free if ALL attendees are free
3. Filter slots to working hours (from settings)
4. Filter slots with duration >= req.DurationMinutes
5. Generate candidate start times (every 30-min interval within free slots)
6. Score each candidate:
    - +40 pts if in morning (09:00–12:00) — mornings are preferred for important meetings
    - +20 pts if in early afternoon (14:00–16:00)
    - -20 pts if it would cut into an existing focus block
    - +15 pts if it's adjacent to another meeting (reduces fragmentation)
    - -10 pts for Friday afternoon (low energy, low availability)
    - +10 pts if it's earlier in the week (Mon–Wed preferred over Thu–Fri for planning meetings)
7. Return top 3 unique slots (no two slots within 1 hour of each other)

`func (e *SmartScheduler) CreateMeeting(ctx, req ScheduleRequest, slot SuggestedSlot) (*calendar.Event, error)`
- Create the Google Calendar event with all attendees
- Write audit_log: action="meeting_scheduled"
