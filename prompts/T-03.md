## Context

The frontend UI is generated separately via Lovable — your task is backend only.

## Task: Settings CRUD API

### storage/settings.go
- `GetSettings(db) (*Settings, error)` — SELECT from settings (always exactly 1 row; auto-insert defaults on first call)
- `SaveSettings(db, s *Settings) error` — UPSERT settings row
- Settings struct fields (must match DB schema exactly):
  WorkStart, WorkEnd, Timezone, FocusMinBlockMinutes, FocusMaxBlockMinutes,
  FocusDailyTargetMinutes, FocusLabel, FocusColor, LunchStart, LunchEnd,
  ProtectLunch, BufferBeforeMinutes, BufferAfterMinutes, CompressionEnabled,
  AutoScheduleEnabled, AutoScheduleCron, LLMProvider, LLMModel, LLMAPIKey, LLMBaseURL

### api/handlers_settings.go
- `GET /api/settings` → return settings as JSON (camelCase json tags)
- `PUT /api/settings` → accept full settings JSON body, validate, save, return updated settings
- Validation rules:
  - WorkStart, WorkEnd, LunchStart, LunchEnd must match HH:MM format
  - *Minutes fields must be positive integers
  - AutoScheduleCron must be a valid 5-part cron expression (use robfig/cron/v3 parser to validate)
  - LLMProvider must be one of: openai, anthropic, ollama
- Return HTTP 400 with `{ "error": "..." }` on validation failure

### CORS
Ensure the CORS middleware (set up in T-01) allows PUT and DELETE methods, and Content-Type: application/json header.
