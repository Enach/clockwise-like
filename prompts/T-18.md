# T-18 — CI/CD pipeline (Docker Hub publish + SSH deploy)

## Goal
On every push to `main`: build and publish backend + frontend + mcp Docker images to Docker Hub, then SSH-deploy to the production server by pulling the new images and restarting containers.

## .github/workflows/sync-and-publish.yml

### Secrets required
| Secret | Purpose |
|---|---|
| `DOCKERHUB_USERNAME` | Docker Hub username for login |
| `DOCKERHUB_TOKEN` | Docker Hub access token |
| `FRONTEND_DEPLOY_KEY` | SSH private key with read access to `Enach/smart-calendar-flow` (used by `webfactory/ssh-agent` so the frontend Dockerfile can `git clone` over SSH) |
| `DEPLOY_HOST` | Production server hostname/IP |
| `DEPLOY_USER` | SSH username on prod server |
| `DEPLOY_SSH_KEY` | SSH private key for prod server |

### Jobs
1. **build-and-push**
   - Logs in to Docker Hub with `DOCKERHUB_USERNAME` + `DOCKERHUB_TOKEN`
   - Sets up SSH agent (`webfactory/ssh-agent`) with `FRONTEND_DEPLOY_KEY`
   - Builds backend image → pushes `enach/paceday-backend:latest` + `:<sha>`
   - Builds frontend image (with `ssh: default`) → pushes `enach/paceday-frontend:latest` + `:<sha>`
   - Builds mcp image → pushes `enach/paceday-mcp:latest` + `:<sha>`
   - Uses GitHub Actions cache (`type=gha`) for layer caching

2. **deploy** (after build-and-push)
   - SSH into prod: `cd /opt/clockwise-like && docker compose -f docker-compose.yml -f docker-compose.prod.yml pull && up -d --remove-orphans && docker image prune -f`

## docker-compose.prod.yml
Overrides services with `image: enach/paceday-{backend,frontend,mcp}:latest` so `docker compose pull` fetches from Docker Hub.

## docker-compose.dev.yml
Exposes `postgres:5432`, `backend:8080`; sets `restart: "no"` on all services for a better local dev experience.

## Frontend Dockerfile change
```dockerfile
# syntax=docker/dockerfile:1
RUN --mount=type=ssh git clone git@github.com:Enach/smart-calendar-flow.git .
```
Requires `ssh-keyscan github.com >> ~/.ssh/known_hosts` before the clone step.
Docker Compose passes the host SSH agent via `ssh: [default]` under the build config.
